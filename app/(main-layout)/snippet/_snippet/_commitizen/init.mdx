---
title: 初始化
---

可在 [官网文档](https://commitizen.github.io/cz-cli/) 了解 `commitizen` 的用法。

安装 `commitizen`：

```bash
npm add -D commitizen
```

配置 `package.json`：

```json
{
  "scripts": {
    "cz": "git-cz"
  }
}
```

`cz` 是脚本名，强烈不推荐改成 `commit`，这个名称会有一些问题；
`git-cz` 是 CLI 命令名，也可以写成 `cz`，两者没有区别，我这里写成 `git-cz` 是为了和键区分开。

配置 `husky`：
可以配置 `prepare-commit-msg` hook，这可以拦截 `git commit`，终端直接弹出 Commitizen 选单；
在 [Husky / Commitizen](/snippet/husky/commitizen) 章节查看如何配置。

进行以上配置后，还需要提供 “适配器”，例如 `cz-conventional-changelog` 等工具；
在 [配置说明](/snippet/commitizen/config) 章节查看如何配置。

配置适配器后，即可通过 `npm run cz` 来使用 Commitizen 的适配器来生成提交消息。
如果也配置了 Husky，那么 `git commit` 时也能触发 Commitizen。
